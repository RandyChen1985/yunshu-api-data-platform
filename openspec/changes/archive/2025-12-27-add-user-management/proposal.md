# Proposal: 用户管理页面

## 概述

为管理后台添加完整的用户管理功能，允许管理员通过 Web 界面进行用户的增删改查操作。

## 动机

当前系统虽然支持基于角色的权限控制，但缺少用户管理界面，管理员需要通过数据库或命令行工具来管理用户，使用体验不佳。需要提供一个直观的用户管理页面，提升管理效率。

## 目标

1. **用户列表展示**：以表格形式展示所有用户信息（用户名、角色、状态、创建时间）
2. **创建用户**：通过表单创建新用户，支持选择角色（admin/user）
3. **编辑用户**：修改用户角色和权限
4. **状态管理**：启用/禁用用户账号
5. **删除用户**：删除不需要的用户
6. **权限控制**：仅管理员可访问此页面

## 设计要点

### 前端实现

- **页面路由**：`/dashboard/users` (已有占位页面)
- **UI 组件**：
  - 用户列表表格（支持搜索、筛选、排序）
  - 创建用户对话框
  - 编辑用户对话框
  - 确认删除对话框
- **状态管理**：使用 Vue 3 响应式状态管理用户数据

### 后端 API

需要实现以下接口：

1. `GET /api/portal/management/users` - 获取用户列表（分页）
2. `POST /api/portal/management/users` - 创建新用户
3. `PUT /api/portal/management/users/{user_id}` - 更新用户信息
4. `DELETE /api/portal/management/users/{user_id}` - 删除用户
5. `PATCH /api/portal/management/users/{user_id}/status` - 切换用户状态

### 权限验证

- 所有用户管理接口需验证调用者为管理员角色
- 普通用户访问返回 403 Forbidden

## 非目标

- 用户密码重置功能（使用 API Key 认证，无需密码）
- 用户自助注册功能
- 批量操作（批量删除、批量禁用等）

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 管理员误删除自己的账号 | 高 | 禁止删除当前登录的管理员账号 |
| 权限验证不完善 | 高 | 在后端统一进行权限检查 |
| API Key 泄露风险 | 中 | 前端仅显示 Key 前缀，完整 Key 仅在创建时展示一次 |
