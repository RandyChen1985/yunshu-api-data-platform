# 提案: Dashboard 24小时请求趋势可视化

## 目标
在 Dashboard 概览页面中增加“近 24 小时请求量趋势图”，帮助用户直观了解系统的活跃度和请求高峰期，补充现有的纯数值统计。

## 背景
目前 Dashboard 仅展示今日、本周、本月的 API 调用总数和成功率等数值。用户无法直观看出过去 24 小时内的波动情况。

## 方案概述
1. **后端**: 扩展或新增 API 接口，从 `api_access_logs` 中按小时聚合近 24 小时的请求量。
2. **前端**: 在 `Overview.vue` 中集成 ECharts 库，并新增一个趋势图表组件（折线图/面积图），展示过去 24 小时每小时的请求分布。

## 详细设计

### 后端 API
- **Endpoint**: `GET /api/portal/dashboard/api-trends-24h`
- **Logic**: 
    - 获取当前时间起向前推 24 小时的起始时间。
    - 按小时（`DATE_FORMAT(created_at, '%Y-%m-%d %H:00:00')`）进行 `GROUP BY`。
    - 返回一个包含 24 个点的数组，即使某小时没有数据也应返回 0，确保图表连续。
- **Response**:
  ```json
  [
    { "timestamp": "2023-10-27 10:00:00", "total_calls": 120, "success_calls": 115 },
    ...
  ]
  ```

### 前端 UI
- **库选择**: `echarts` + `vue-echarts`。
- **组件**: 在“系统概览”或“我的工作台”首屏下方增加一个全宽的卡片。
- **交互**: 支持鼠标悬停显示具体小时的统计数据。

## 风险与挑战
- **性能**: `api_access_logs` 表记录可能非常多，按小时聚合需要索引支持。建议确保 `created_at` 有索引。
- **前端打包**: 引入 ECharts 会增加前端打包体积，考虑按需引入关键模块。

## 结论
该改动将显著提升 Dashboard 的实用性和视觉体验，且实现路径清晰，风险可控。
