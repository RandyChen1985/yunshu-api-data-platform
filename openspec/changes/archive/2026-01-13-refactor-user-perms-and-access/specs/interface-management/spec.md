# 接口管理规格变更 (Spec Delta)

## MODIFIED Requirements

### Requirement: 接口管理访问权限
系统 **必须 (MUST)** 允许用户根据其角色查看“接口管理”（资源）模块。
-   **已修改**：所有经过身份验证的用户（管理员和普通用户） **必须 (MUST)** 能够访问“接口管理”菜单和列表视图。
-   **新增**：普通用户 **必须 (MUST)** 拥有 **只读** 权限。他们 **不得 (MUST NOT)** 创建、编辑、删除资源或切换资源状态。
-   **未变更**：管理员保留完全控制权（增删改查）。

#### Scenario: 普通用户查看资源列表
-   **假设** 我是一个已登录的普通用户
-   **当** 我导航到“接口管理”页面
-   **那么** 我可以看到资源列表
-   **但是** 我看不到“新建资源”、“编辑”或“删除”按钮
-   **并且** “状态”切换开关处于禁用状态

#### Scenario: 普通用户查看资源详情
-   **假设** 我是一个已登录的普通用户
-   **当** 我点击某个资源查看详情时
-   **那么** 我可以看到资源配置表单
-   **但是** 所有字段都处于禁用状态（只读）
-   **并且** 没有“保存”按钮

#### Scenario: 后端权限强制执行
-   **假设** 我是一个已登录的普通用户
-   **当** 我尝试向 `/api/portal/meta/resources` 发送 POST/PUT/DELETE 请求时
-   **那么** 系统将返回 403 Forbidden 错误
