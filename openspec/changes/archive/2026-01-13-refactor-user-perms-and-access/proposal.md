# 重构用户权限分配与接口访问控制

**变更 ID**: `refactor-user-perms-and-access`

## 概述
本提案旨在通过将权限分配组织到用户弹窗内的专用标签页中，优化用户管理 UI。同时，授予普通用户对“接口管理”（资源）模块的只读访问权限，允许他们查看可用资源而无需修改权限。

## 动机
-   **易用性**：目前的用户创建/编辑弹窗比较拥挤。将权限移至独立标签页可提高可读性和管理效率。
-   **透明度**：普通用户应当能够查看可用接口（资源）列表，以了解系统中潜在可用的数据，即使他们不能修改定义。

## 核心变更
1.  **用户管理 UI**：重构创建/编辑用户弹窗，使用“基础信息”和“权限分配”两个标签页。
2.  **接口管理访问**：
    -   对所有用户开放“接口管理”菜单。
    -   在接口管理列表和详情页中，对非管理员用户强制执行只读访问。
    -   更新后端权限，允许非管理员列出/查看资源，但禁止创建/编辑/删除操作。
