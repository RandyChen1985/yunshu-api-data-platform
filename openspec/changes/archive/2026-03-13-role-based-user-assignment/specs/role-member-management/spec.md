# 角色成员管理能力规范 (Role Member Management Capability Spec)

本规范定义了从角色中心视角管理用户分配的行为。

## 1. 用户界面：角色成员管理对话框

### 需求：角色成员选择与分配
系统 **必须 (MUST)** 提供一种在角色管理 UI 中单次操作即可将多个用户分配给角色的方法。

#### 场景：将现有用户分配给角色
- **假设 (GIVEN)**：管理员处于角色管理页面。
- **当 (WHEN)**：他们点击特定角色的“成员管理”按钮。
- **那么 (THEN)**：弹出一个显示两组用户列表的对话框：
  - **备选用户**：系统中所有当前未属于该角色的活跃用户。
  - **已选用户**：当前已分配给该角色的所有用户。
- **当 (WHEN)**：管理员在列表间移动用户并点击“保存”。
- **那么 (THEN)**：系统 **应当 (SHALL)** 更新角色-用户关系，使其与“已选用户”列表匹配。

## 2. API 接口规范

### 接口：GET `/api/portal/management/roles/{role_id}/users`
返回当前分配给指定角色的所有用户的 ID。

- **成功响应**：`BaseResponse`，`data` 为用户 ID 列表。
- **权限要求**：需要 `admin` 角色。

### 接口：PUT `/api/portal/management/roles/{role_id}/users`
更新指定角色的成员关系。

- **请求体 (Request Body)**：
  ```json
  {
    "user_ids": [1, 2, 3, ...]
  }
  ```
- **行为逻辑**：
  - 系统 **应当 (SHALL)** 将指定 `role_id` 的所有现有用户关联替换为提供的 `user_ids`。
  - 系统 **应当 (SHALL)** 使所有受影响用户（新增和移除的用户）的权限缓存失效。
- **权限要求**：需要 `admin` 角色。

## 3. 约束与校验
- **无效 ID 处理**：如果提供的任何 `user_id` 不存在，系统 **应当 (SHALL)** 返回 400 错误且不应用任何变更。
- **仅限管理员访问**：仅具有管理员权限的用户才允许管理角色成员关系。
