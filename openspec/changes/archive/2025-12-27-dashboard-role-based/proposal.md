# Dashboard 页面角色差异化展示

## 背景

当前系统的 Dashboard 首页（`/dashboard`）仅显示简单的占位文本，缺少实际内容和功能。需要开发一个完整的 Dashboard 页面，根据用户角色（管理员/普通用户）展示不同的内容和统计数据。

## 目标

1. **管理员 Dashboard**：展示全局统计数据、系统概览、快捷操作
2. **普通用户 Dashboard**：展示个人使用数据、API 使用统计、快速开始指南
3. **响应式设计**：适配不同屏幕尺寸
4. **数据实时性**：统计数据实时从后端获取

## 范围

### 管理员 Dashboard 内容

1. **统计卡片**（4个）
   - 总用户数
   - 活跃用户数（最近7天）
   - 总 API 调用次数（今日/本周/本月）
   - 系统健康状态

2. **图表展示**
   - API 调用趋势图（最近7天/30天）
   - 用户活跃度分布
   - 错误率趋势

3. **快捷操作**
   - 创建新用户
   - 查看审计日志
   - 系统设置

4. **最近活动**
   - 最新用户注册
   - 最近 API 调用记录
   - 系统告警（如有）

### 普通用户 Dashboard 内容

1. **个人统计卡片**（4个）
   - 我的 API Key 状态
   - 我的 API 调用次数（今日/本周/本月）
   - 平均响应时间
   - 成功率

2. **快速开始**
   - API 文档链接
   - 示例代码
   - API 调试入口

3. **我的最近调用**
   - 最近调用记录（时间、接口、状态）

4. **使用指南**
   - 如何获取 API Key
   - 常用接口说明
   - 注意事项

## 技术方案

### 后端 API

需要新增以下统计接口：

1. **管理员统计接口** `GET /api/portal/dashboard/admin-stats`
   - 返回：总用户数、活跃用户数、API 调用统计、错误率等

2. **用户统计接口** `GET /api/portal/dashboard/user-stats`
   - 返回：个人 API 调用统计、成功率、平均响应时间等

3. **API 调用趋势接口** `GET /api/portal/dashboard/api-trends`
   - 参数：`days` (7/30)
   - 返回：每日调用量、成功率、错误数等

4. **最近活动接口** `GET /api/portal/dashboard/recent-activities`
   - 返回：最近用户注册、API 调用记录等

### 前端实现

1. **创建 `Overview.vue` 组件**
   - 根据 `userInfo.role` 动态渲染不同内容
   - 使用 `v-if` 区分管理员和普通用户视图

2. **统计卡片组件**
   - 可复用的卡片组件
   - 支持图标、数值、趋势箭头

3. **图表组件**
   - 使用轻量级图表库（如 Chart.js 或 Apache ECharts）
   - 响应式设计

4. **数据刷新**
   - 页面加载时获取数据
   - 提供手动刷新按钮
   - 可选：定时自动刷新（每 30 秒）

## 交付物

1. 后端新增 4 个统计接口
2. 前端 `Overview.vue` 组件
3. 自动化测试（后端接口测试）
4. 更新路由配置

## 优先级

高优先级 - Dashboard 是用户登录后的第一个页面，是系统的门面。

## 时间估算

- 后端 API 开发：2-3 小时
- 前端组件开发：3-4 小时
- 测试和调试：1-2 小时
- **总计**：6-9 小时
